# Kimi K2模型简介

Kimi K2是一款采用MoE架构的大语言模型（LLM），面向通用对话、代码生成与Agent任务进行优化。
模型在容量与推理成本之间做了平衡设计，采用稀疏激活路径提升参数利用率，并强化了复杂任务场景下的稳定性与可用性。
与DeepSeek V3相比，Kimi K2在参数规模上更大（Kimi K2总参数约1T，DeepSeek V3约671B），但单token激活参数更低（Kimi K2约32B，DeepSeek V3约37B），因此在保持模型容量的同时更强调稀疏计算效率。

效果：在代码、推理与工具调用相关评测中，Kimi K2表现出较强竞争力，适用于真实业务中的多步任务处理。

性能：通过注意力与专家路由协同优化，Kimi K2在保持模型能力的同时降低了在线推理开销，适配高并发与长上下文场景。

架构特点：

- 采用MoE架构，按token动态路由激活少量专家，提升计算效率
- 注意力模块采用MLA，降低KV缓存占用并优化长序列推理成本
- 支持较长上下文窗口，适用于文档级理解与多轮交互任务
- 面向工程部署优化吞吐与稳定性，兼顾离线与在线服务需求

## 整体架构



整体架构由Embedding层、MLA注意力层、MoE前馈层和输出层组成。核心思路是将高效注意力计算与稀疏专家计算结合：前者用于缓解长上下文下的显存与计算压力，后者用于在可控成本下提升模型容量与表达能力。

参数对比（表格形态参考Kimi K2官方模型卡片）：


| **参数项**         | **Kimi K2** | **DeepSeek V3** |
| --------------- | ----------- | --------------- |
| **架构**          | MoE         | MoE             |
| **总参数量**        | 1T          | 671B            |
| **激活参数量**       | 32B         | 37B             |
| **层数**（含稠密层）    | 61          | 61              |
| **稠密层数量**       | 1           | 3               |
| **注意力头数**       | 64          | 128             |
| **专家数量**        | 384         | 256             |
| **每token选择专家数** | 8           | 8               |
| **共享专家数量**      | 1           | 1               |
| **词表大小**        | 160K        | 129K            |
| **上下文长度**       | 128K        | 128K            |
| **注意力机制**       | MLA         | MLA             |


从工程视角看，Kimi K2的架构特点主要体现在三点：其一，MLA改善长序列推理效率；其二，MoE按token路由激活少量专家，提升单位计算收益；其三，训练与推理链路协同优化，支持大规模部署下的稳定吞吐。
